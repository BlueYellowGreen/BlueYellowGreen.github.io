TIP

윈도우에서 윈도우 명령어와 리눅스 명령어를 같이 사용하는 터미널을 사용하고 싶다면, cmder (opens new window)를 사용하면 된다.

윈도우 vs 리눅스 주요 명령어

CD   ↔   cd

CLS   ↔   clear

COPY   ↔   cp

DEL   ↔   rm

DIR   ↔   ls


  1. 파이썬은 인터프리터 언어로, 플랫폼에 독립적이다. 소스코드 실행 시, 인터프리터가 운영체제에 맞게 읽어 주기 때문에 동적 타이핑 언어이기도 하다.

  2. 폰 노이만 구조 (opens new window)를 가지는 컴퓨터의 특징 상, 변수는 메모리 주소를 가지고 있고, 값은 메모리 주소에 할당된다.

  3. 이진수 한 자리를 bit 라고 칭하고 8 bit는 1 byte를 의미한다. (int ➜ 4 byte)

  4. 파이썬에서 is 는 메모리 주소를 비교하는 연산자이다. 다만, 파이썬에서 -5 ~ 256 범위의 숫자는 미리 메모리에 할당을 해 놓아, 같은 주소를 가리키게 된다.

  5. 파이썬에서 객체의 메모리 공간 사용량을 확인하려면 다음의 방식으로 사용하면 된다.

import sys
print(sys.getsizeof("이 문자열의 메모리 공간 사용량은?"))

  1. 컴퓨터는 문자열 그 자체를 메모리에 저장하지 않고, 2진수로 바꿔 저장한다. (모든 것을 2진수로 바꿔 저장한다.) 바꾸는 표준 규칙은 UTF-8 (opens new window)을 따른다.

  2. 파이썬에서 Escape Character까지 포함한 문자열을 얻으려면 raw string이라 불리는 r 을 사용하면 된다. ➜ r'이렇게 쓰면\n된다.'

  3. 파이썬은 객체 참조에 의한 호출 (call by object reference) 방식을 따르므로, 함수로 전달된 객체를 참조하여 변경할 경우 영향을 주지만, 새로운 객체를 만들 경우 영향을 주지 않는다.

def is_this_change(i_am_object):
    i_am_object.append("me too") # 원본 변경 O
    i_am_object = ["You don't change as this way."] # 원본 변경 X

lets_check_object = ["I am object!"]

is_this_change(lets_check_object)

lets_check_object # ["I am object!", "me too"]

  1. 파이썬은 3.5 이후 부터 PEP-484에 기반하여 type hints를 제공한다. 충실히 작성하면 모두에게 좋다.

  2. 파이썬에서 docstring을 상세히 작성하면, 마찬가지로 해당 함수 작성 시 매우 유용하다.
